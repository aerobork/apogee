Overview

Welcome to the documentation for apogee.js! This is a javascript library for simulating model rockets. This section gives a brief overview of how apogee.js functions, as well as useful information like coordinate systems and naming conventions.


Common Variables

we only use two variables in the entire library: state and points, there are no other variables.


Abbreviations

Rockets depend on a few variables during flight, like angle of attack and velocity. In general apogee.js will name its variables as shown below:

angle of attack = angle
reference area = aref
reference diameter = dref
free-stream velocity = v0
rho = p
mach number = M
coefficient of normal force = n
coefficient of pitch moment = n
coefficient of drag = cd
coefficient of lift = dl
center of gravity = cg
center of pressure = cp

Coordinate System

THe origin is placed at the tip of the nosecone, with the positive y-axis on the rocket's axis of rotation. The x-axis is perpendicular to the axis of rotation.

Components

Components are the basic buildings blocks of any rocket in apogee.js. Components are categorized into three major component types: Outer Components, Inner Components, and Component Series. All three of these components classes inherit the Component superclass. All component classes contain a state variable, which represents anything that is not a derived property of that component.

For example, if the length of a component is not needed to specify the shape of a component, but is used in a calculation at some point, it may be derived using the state variables. Because of this, length will not be in the state, but instead will be an internally accessible variable this.length

class Component(state)

Base component. This should not be used in most circumstances, unless you are extending it to create a new class.
